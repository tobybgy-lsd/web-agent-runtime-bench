# GitHub Discussions Setup

Discussions are enabled for this repository.

Current GitHub-provided categories include:

- Ideas
- Show and tell

The intended community categories are:

- Failure Cases
- Ideas
- Show and Tell

`Failure Cases` is the important missing category for collecting sanitized
failure samples from users. GitHub CLI and GraphQL currently expose creating
discussions, but not creating repository discussion categories, so this category
must be added manually in the GitHub repository settings UI.

Recommended description:

```text
Share sanitized AI automation, Playwright, browser workflow, RPA, or browser-agent
failure cases for diagnosis and possible inclusion in the public validation
corpus. Do not include secrets, cookies, tokens, credentials, private
screenshots, or personal data.
```
